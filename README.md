# Linux系统编程 - 多用户聊天系统

这是一个基于C语言开发的多用户聊天服务器和客户端程序，支持用户认证（注册/登录）、消息广播等功能。

## 功能特性

- **多用户支持**：最多支持10个并发客户端连接
- **用户认证**：支持用户注册和登录功能
- **安全存储**：用户密码采用SHA-256哈希加密存储
- **实时消息广播**：用户发送的消息会广播给其他在线用户
- **线程安全**：使用互斥锁保证多线程环境下的数据安全

## 系统架构

### 服务端 (chat_server.c)
- 使用TCP套接字实现网络通信
- 支持多线程处理多个客户端连接
- 实现了用户认证系统
- 使用互斥锁保证线程安全
- 数据持久化存储到配置文件

### 客户端 (chat_client.c)
- 简单的命令行界面
- 支持多线程收发消息
- 消息格式化显示

## 编译和运行

### 编译服务端
```bash
gcc chat_server.c -o chat_server -lssl -lcrypto -lpthread
```

### 编译客户端
```bash
gcc chat_client.c -o chat_client -lpthread
```

### 运行服务端
```bash
./chat_server <端口号>
```

例如：
```bash
./chat_server 8080
```

### 运行客户端
```bash
./chat_client <服务器IP地址> <端口号>
```

例如：
```bash
./chat_client 127.0.0.1 8080
```

## 用户认证流程

首次连接时，客户端会被要求选择注册或登录：

- 输入 `register` 进入注册流程
- 输入其他内容将进入登录流程

### 注册流程
1. 输入用户名
2. 输入密码
3. 系统检查用户名是否唯一
4. 密码经过SHA-256哈希后存储

### 登录流程
1. 输入用户名
2. 输入密码
3. 系统验证凭据

## 消息交互

- 客户端发送消息后，会广播给其他所有连接的客户端
- 消息会显示为 `[用户名]: 消息内容` 的格式
- 在客户端输入 `/exit` 可以断开连接并退出

## 数据存储

- 用户信息存储在 [users.conf](file:///d:/linux_jiangjian/linux-system-programming/sourse/users.conf) 文件中
- 格式为 `用户名:SHA-256哈希密码`，每行一个用户
- 服务端启动时自动加载用户数据

## 技术要点

- **套接字编程**：使用TCP协议进行可靠通信
- **多线程**：每个客户端连接由独立线程处理
- **加密哈希**：使用OpenSSL库的SHA-256算法保护用户密码
- **同步机制**：使用互斥锁防止竞态条件
- **内存管理**：合理分配和释放资源

## 依赖库

- `libc` - 标准C库
- `libssl/libcrypto` - OpenSSL加密库，用于SHA-256哈希
- `libpthread` - POSIX线程库，用于多线程支持

## 安全特性

- 密码不会以明文形式存储，而是使用SHA-256哈希
- 多线程环境下的数据访问通过互斥锁保护
- 用户名唯一性检查防止重复注册

## 限制

- 最大支持10个并发客户端
- 用户名最大长度32字符
- 缓冲区大小限制为1024字节
- 未实现消息加密传输（仅密码存储加密）

## 故障排除

1. **编译错误**：确认已安装OpenSSL开发库
2. **连接失败**：检查服务器IP地址和端口号是否正确
3. **权限问题**：避免使用需要特权的低编号端口（< 1024）
