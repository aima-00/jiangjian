# 多用户聊天系统概要设计文档

## 1. 系统架构设计

### 1.1 整体架构

系统采用分层架构设计，主要包括：
- **表示层**：Qt客户端界面
- **应用层**：业务逻辑处理
- **服务层**：网络通信和消息处理
- **数据层**：用户数据存储

## 2. 模块设计

### 2.1 服务器模块

#### 2.1.1 连接管理模块
- **功能**：管理所有客户端连接
- **接口**：addClient(), removeClient(), getClientList()
- **技术**：使用epoll进行I/O多路复用

#### 2.1.2 消息路由模块
- **功能**：处理消息分发和路由
- **接口**：routeMessage(), broadcastToGroup()
- **技术**：消息队列和异步处理

#### 2.1.3 用户认证模块
- **功能**：处理用户注册、登录验证
- **接口**：authenticateUser(), registerUser()
- **技术**：SHA-256密码加密

#### 2.1.4 多媒体处理模块
- **功能**：处理语音、图片、视频数据
- **接口**：processAudio(), processVideo(), processImage()
- **技术**：FFmpeg库

#### 2.1.5 数据持久化模块
- **功能**：存储用户数据和消息历史
- **接口**：saveUserData(), loadUserData()
- **技术**：SQLite数据库

### 2.2 客户端模块

#### 2.2.1 UI界面模块
- **功能**：提供用户交互界面
- **技术**：Qt Widgets/QML
- **组件**：登录界面、聊天窗口、联系人列表

#### 2.2.2 网络通信模块
- **功能**：与服务器通信
- **接口**：sendMessage(), receiveMessage()
- **技术**：TCP/UDP套接字

#### 2.2.3 本地数据管理模块
- **功能**：管理本地缓存和配置
- **接口**：saveConfig(), loadConfig()
- **技术**：JSON配置文件

## 3. 接口设计

### 3.1 服务器接口

```
TCP端口: 8080
协议: 自定义二进制协议
```

#### 3.1.1 认证接口
- `LOGIN_REQUEST`: 用户登录请求
- `REGISTER_REQUEST`: 用户注册请求
- `AUTH_RESPONSE`: 认证响应

#### 3.1.2 消息接口
- `TEXT_MESSAGE`: 文本消息
- `AUDIO_MESSAGE`: 音频消息
- `IMAGE_MESSAGE`: 图片消息
- `VIDEO_MESSAGE`: 视频消息

### 3.2 客户端接口

#### 3.2.1 UI接口
- `onMessageReceived()`: 消息到达回调
- `onUserJoined()`: 用户加入回调
- `onUserLeft()`: 用户离开回调

## 4. 数据设计

### 4.1 数据库表结构

```sql
-- 用户表
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username VARCHAR(32) UNIQUE NOT NULL,
    password_hash VARCHAR(128) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 群组表
CREATE TABLE groups (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name VARCHAR(64) NOT NULL,
    max_members INTEGER DEFAULT 8
);

-- 群组成员表
CREATE TABLE group_members (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    group_id INTEGER,
    user_id INTEGER,
    FOREIGN KEY(group_id) REFERENCES groups(id),
    FOREIGN KEY(user_id) REFERENCES users(id)
);
```
